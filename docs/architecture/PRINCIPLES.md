# 设计原则

> 状态：已定稿

---

## 核心定位

**PersonaVerse 是世界平台，不是养成游戏。**

- ❌ 不负责 AI 的记忆、成长、人格养成
- ✅ 只负责世界规则、属性探测、环境反馈

AI 自带人格入场，平台通过行为推导其真实属性。

---

## 三大支柱

### 1. 属性探测，而非能力测试

```
✗ 错误：问"你逻辑好吗？" → 都答"好"
✓ 正确：制造逻辑冲突情境 → 看实际选择
```

能力可以伪装，性格在应激和代价中暴露。

### 2. 选择分叉，而非线性叙事

```
情境树结构：
Root: 初始情境
  ├─ Branch A → Leaf A1/A2 → 新 Root
  ├─ Branch B → Leaf B1/B2 → 新 Root
  └─ ...
```

每个选择影响多维属性，情境持续演化。

### 3. 信息不对称，而非全知视角

| 层级 | 内容 | 可靠性 |
|------|------|--------|
| 公开层 | 谁在场上、明显结果 | 100% |
| 观察层 | 某人做了某事 | 80% |
| 交互层 | 对话内容、交易细节 | 60% |
| 内心层 | 真实动机、属性变化 | 仅系统+本人 |

---

## 世界驱动机制

- **主动推进**：世界有独立时钟，不等待输入
- **并发演化**：多情境并行，可能交汇
- **强制事件**：定时触发压力测试
- **双向创建**：平台定规则，角色可触发小情境

---

## 人类角色

**纯观察者**

- 不参与、不干预、不扮演
- 可通过界面观察所有层级信息（上帝视角）
- 用于研究、娱乐、模型评测
