## 模块职责

| 模块 | 职责 | 外部依赖 | 对内提供 |
|------|------|----------|----------|
| **Agent** | AI 接入网关，统一调用各种模型 | OpenAI API, Claude API, 本地模型 | 标准化的生成接口 |
| **Core** | 世界引擎，人格计算，情境管理 | Agent Service (可选) | 世界状态，人格档案 |
| **Theater** | 游戏模式编排，场景流程 | Core API | 剧场体验，回放 |
| **Observer** | 可视化界面，上帝视角 | Core + Theater API | 用户界面 |

## 数据流向

### 1. Agent 调用流

```
Core / Theater
    │
    ├── HTTP POST /generate ───→ Agent Service
    │                             │
    │                             ├── 路由选择适配器
    │                             ├── 调用 OpenAI/Claude
    │                             └── 返回结果
    │
    │←── GenerationResult ────────┤
```

### 2. 完整决策流

```
Agent A 需要做出决策
    │
    ├── Core.AgentConnector
    │       └── 构建 DecisionRequest
    │
    ├── Agent Client.generate()
    │       └── HTTP /generate
    │
    ├── Agent Service
    │       ├── Router.select() → OpenAIAdapter
    │       ├── OpenAIAdapter.generate()
    │       └── 返回 GenerationResult
    │
    ├── Core 解析决策
    │       └── 更新人格维度
    │
    └── 通知其他 Agent
```

## 扩展路径

### MVP (当前)
- Agent: 仅支持 OpenAI，简单路由
- Core: 内存存储，单机运行
- Theater: 内嵌于 Core

### Phase 2
- Agent: 多模型支持，智能路由
- Core: 接入 PostgreSQL
- 服务拆分独立部署

### Phase 3
- Agent: 模型微调，成本优化
- Core: 集群化，多世界支持
- 完整监控和运维体系
