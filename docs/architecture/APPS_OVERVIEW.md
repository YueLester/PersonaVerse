## 职责边界

| 模块 | 负责 | 不负责 |
|------|------|--------|
| **Core** | 世界时钟、Agent管理、人格计算、情境执行、信息分发 | 游戏流程编排、模式特定规则、回放生成 |
| **Theater** | 模式定义、场景编排、匹配系统、模式评分、回放生成 | Agent状态管理、人格维度计算、基础世界规则 |
| **Observer** | UI展示、实时流接收、用户交互、可视化 | 任何业务逻辑、状态变更、决策计算 |
| **SDK** | 协议封装、事件处理、重连逻辑 | 决策逻辑（由接入方实现） |

## 数据流向

### 1. Agent决策流

```
Agent (外部)
    │
    ├── 通过SDK接收决策请求
    │
    ▼
Core.WorldEngine
    │
    ├── 收集到所有参与者决策
    │
    ▼
Core.ScenarioRunner.resolve_choices()
    │
    ├── 计算结果
    ├── 调用 PersonalityEngine 更新属性
    │
    ▼
Core.EventBus
    │
    ├── 推送结果给Agent
    ├── 推送更新给Observer
    │
    ▼
Theater (如有关联) 接收事件更新舞台状态
```

### 2. 观察者数据流

```
Observer 打开界面
    │
    ├── HTTP GET /world/state (初始数据)
    ├── WS /ws/connect (建立连接)
    ├── WS subscribe: world (订阅世界频道)
    │
    ▼
持续接收:
    • agent.update    → 更新Agent卡片
    • scenario.advance → 更新情境树
    • event.new       → 添加到事件流
    • theater.state   → 更新剧场舞台
```

## 扩展路径

### 短期（MVP）
- Core单实例
- Theater内嵌于Core（简化部署）
- Observer直连Core

### 中期
- Theater独立部署
- Core支持多实例（通过Redis协调）
- 增加Analyzer模块（数据分析）

### 长期
- Core集群化
- 多世界支持（World Sharding）
- UGC场景编辑器
